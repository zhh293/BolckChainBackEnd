# 错误码

本文档列出了API接口可能返回的所有错误码及其含义。

## 错误码列表

### 通用错误码

| 错误码 | 错误信息 | 说明 | 解决方案 |
|--------|----------|------|----------|
| 400 | 请求参数错误 | 请求参数不正确或缺少必填参数 | 检查请求参数是否符合接口要求 |
| 401 | 未授权 | 未找到有效 Session，浏览器未携带 JSESSIONID 或 Session 已失效 | 重新登录获取新的Session ID |
| 404 | 资源不存在 | 请求的资源不存在 | 检查资源ID是否正确 |
| 500 | 服务器内部错误 | 服务器处理请求时发生错误 | 联系技术支持 |

## 错误响应格式

所有错误响应都遵循统一的格式：

```json
{
  "code": 400,
  "message": "错误信息",
  "data": null
}
```

### 字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | integer | 错误码 |
| message | string | 错误信息 |
| data | object/null | 附加数据（通常为null） |

## 错误处理建议

### 客户端错误处理

1. **检查HTTP状态码**: 首先检查HTTP状态码，确定请求是否成功
2. **解析错误响应**: 如果状态码不是2xx，解析错误响应中的code和message字段
3. **显示友好提示**: 根据错误码和错误信息，向用户显示友好的错误提示
4. **重试机制**: 对于网络错误或服务器错误，可以实现自动重试机制

### 错误处理示例

```javascript
async function callApi(url, options) {
  try {
    const response = await fetch(url, options);
    
    if (!response.ok) {
      const error = await response.json();
      throw new Error(error.message || '请求失败');
    }
    
    return await response.json();
  } catch (error) {
    console.error('API调用失败:', error);
    throw error;
  }
}
```

## 常见问题

### Q: 如何处理Session过期？

A: 当收到401错误时，应该引导用户重新登录，获取新的Session ID。

### Q: 如何处理网络错误？

A: 网络错误通常是临时性的，可以实现自动重试机制。建议最多重试3次，每次间隔递增。

### Q: 如何处理服务器错误（500）？

A: 服务器错误通常是系统异常，应该记录错误日志，并向用户显示友好的错误提示，建议用户稍后重试。

### Q: 错误码会变化吗？

A: 错误码是稳定的，不会随意变化。但新增错误码时，会在更新日志中说明。
