# 认证管理

认证管理模块提供管理员登录、登出等功能。

## 认证机制

本API使用Session认证机制。管理员登录后，系统会返回一个Session ID，后续请求需要在Cookie中携带此Session ID。

### Session配置

- **Session名称**: `JSESSIONID`
- **有效期**: 1小时（3600000毫秒）
- **存储方式**: Cookie

### 认证流程

1. 管理员调用登录接口，提交用户名和密码
2. 服务器验证用户名密码，验证成功后创建Session
3. 服务器返回Session ID，并设置到Cookie中
4. 后续请求自动携带Cookie中的Session ID
5. 服务器验证Session ID的有效性，验证通过后处理请求

## 接口列表

### 1. 管理员登录

**接口描述**: 管理员登录接口，验证用户名密码

**请求方式**: `POST`

**请求路径**: `/auth/login`

**请求头**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Content-Type | string | 是 | application/json |

**请求体**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名（3-50字符） |
| password | string | 是 | 密码（6-100字符） |

**请求示例**:

```json
{
  "username": "admin",
  "password": "123456"
}
```

**成功响应**:

| 状态码 | 说明 |
|--------|------|
| 200 | 登录成功 |

**响应体**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| token | string | 会话标识 |
| refreshToken | string | 刷新令牌 |
| expiresIn | integer | 过期时间（毫秒） |
| username | string | 用户名 |
| role | string | 角色 |

**响应示例**:

```json
{
  "token": "admin_session",
  "refreshToken": "",
  "expiresIn": 3600000,
  "username": "admin",
  "role": "ADMIN"
}
```

**失败响应**:

| 状态码 | 说明 |
|--------|------|
| 400 | 用户名或密码错误 |

**错误响应示例**:

```json
{
  "message": "用户名或密码错误"
}
```

### 2. 检查登录状态

**接口描述**: 检查管理员是否已登录

**请求方式**: `GET`

**请求路径**: `/auth/validate`

**请求头**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Cookie | string | 是 | JSESSIONID=xxx |

**成功响应**:

| 状态码 | 说明 |
|--------|------|
| 200 | 登录状态 |

**响应体**:

| 类型 | 说明 |
|------|------|
| boolean | true表示已登录，false表示未登录 |

**响应示例**:

```json
true
```

**接口描述**: 管理员登出接口

**请求头**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Cookie | string | 是 | JSESSIONID=xxx |

**成功响应**:

| 状态码 | 说明 |
|--------|------|

**响应示例**:


## 注意事项

1. **Session有效期**: Session默认有效期为1小时，过期后需要重新登录
2. **并发登录**: 同一账号可以同时登录多个设备
3. **安全性**: 请妥善保管用户名和密码，不要在客户端代码中硬编码
4. **登出操作**: 建议在用户退出应用时调用登出接口，释放服务器资源

## 常见问题

### Q: Session过期后如何处理？

A: Session过期后，需要重新调用登录接口获取新的Session ID。

### Q: 如何判断用户是否已登录？

A: 调用`/auth/validate`接口，返回true表示已登录，false表示未登录。

### Q: 登录失败的原因有哪些？

A: 常见原因包括：
- 用户名或密码错误
- 账号被禁用
- 网络连接问题

### Q: 如何实现"记住我"功能？

A: 可以使用refreshToken机制，在Session过期时使用refreshToken获取新的Session。
